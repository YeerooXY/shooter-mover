# DEMO-006 — Integrated feature proof

## Objective

Create the next visible demo composition after all dependencies are accepted. This task is the sole owner of final demo scene/controller composition.

## Required proof

- Character/class selection reaches combat.
- Player uses an approved real weapon and can boost.
- Moving droid uses its supplied sprite, can be damaged, dies, and contributes XP/reward state.
- Turret attacks according to its facing/angle rules and can be destroyed.
- A destructible prop can drop a physical strongbox or scrap.
- The supplied open door transitions between two rooms.
- Map state records the two-room route.
- One activated ability works.
- One skill and one augment visibly affect runtime behavior.
- A crafted weapon enters inventory and can be equipped.
- End Run reaches Results with unopened strongbox instances.

## Validation

Cold Unity compile, focused EditMode/PlayMode XML, and a manual end-to-end capture. This task must not silently replace or bypass the existing reward, inventory, equipment, XP, or exact-instance authorities.
